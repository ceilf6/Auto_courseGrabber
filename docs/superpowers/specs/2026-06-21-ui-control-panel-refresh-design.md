# UI 控制面板视觉刷新设计

关联 Issue：[#20](https://github.com/ceilf6/Auto_courseGrabber/issues/20)

## 目标

将 `courseGrabber.js` 注入的控制面板从通用紫色渐变样式调整为信息优先的深色「考场控制台」。改动必须让课程配置、运行状态和日志更易扫描，同时保持既有自动选课功能和调用接口不变。

## 设计方向

面板以深海军蓝为基底，避免与常见教务页面的浅色内容混淆：

- 青绿色表示运行、成功和主操作；
- 琥珀色表示定时、提示与等待；
- 朱红色只表示停止、错误和破坏性操作；
- 低饱和蓝灰用于分组边界、次级按钮和辅助信息。

标题栏保留拖拽区域，但改为紧凑的应用栏；内容区使用清晰的卡片分组和紧凑网格。日志保持等宽字体，形成与表单区不同的信息层级。

## 范围与边界

仅修改 `createUI()` 中的 CSS，以及为表现层级所需的静态 HTML 标记。不会修改选课、定时、课程管理、拖拽、日志拦截或事件绑定的 JavaScript 行为。

以下 DOM 合约必须保持不变：

- `courseGrabberUI`、`cg-status-text`、`cg-course-list`、`cg-log-area` 和所有表单/按钮 ID；
- `cg-header` 拖拽选择器，以及动态渲染课程列表所使用的 `cg-*` 类；
- `window.showGrabberUI`、`window.editCourseUI`、`window.removeCourseUI` 和既有点击处理程序；
- `TARGET_COURSES` 的数据格式与运行状态更新方式。

不新增包、构建步骤、远程字体或图片资源。

## 组件设计

### 面板与标题栏

- 维持右上角固定定位、可拖拽、可最小化和可关闭的交互。
- 使用不透明深色表面、细边框和分层阴影，替代全局渐变及玻璃模糊效果。
- 标题文字使用本机可用的中文优先字体栈；不依赖网络字体。

### 状态、表单和课程列表

- 状态采用高对比度徽标和呼吸指示点，运行/空闲状态有明确颜色区分。
- 输入框在默认、悬停、聚焦和禁用状态均有可辨识边界；`focus-visible` 为键盘用户提供可见焦点。
- 课程清单提升课程号、优先级和筛选条件的视觉层级，编辑和删除操作保持原有 ID/全局函数调用方式。

### 定时、操作与日志

- 定时器以琥珀色强调，倒计时仍沿用原有动态更新逻辑。
- 开始操作为唯一高权重主按钮；停止操作持续为朱红色；次级动作降低视觉权重。
- 日志区使用深色等宽背景和类型色，不改变日志条目的 class 或追加行为。

### 响应式与动效

- 在窄视窗下将面板宽度限制为可视区域，双列按钮仍保持可点按。
- 动画只限状态点和倒计时等反馈；`prefers-reduced-motion: reduce` 会禁用非必要动画和过渡。

## 验证策略

1. 用 `node --check courseGrabber.js` 验证脚本语法。
2. 静态核对所有绑定事件依赖的 DOM ID、关键 `cg-*` class 和全局 UI 函数未被移除。
3. 在浏览器夹具中验证默认、运行、课程列表、定时、最小化和窄屏呈现；同时确认控制台没有脚本错误。
4. 运行 `git diff --check`，确认补丁没有空白错误。

## 非目标

- 不改变业务流程、输入文案含义、功能可用性或任何自动选课策略。
- 不引入主题切换、持久化设置、新表单字段或新的用户操作。
